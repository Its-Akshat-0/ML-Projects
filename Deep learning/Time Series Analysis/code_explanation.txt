# Code Explanation: Time Series Analysis Project

This document provides a step-by-step explanation of the code implemented in `time_series_analysis.ipynb`.

## 1. Data Loading and Preprocessing
- **Library**: `pandas`
- **Action**: 
    - The dataset `stock_price.csv` is loaded into a DataFrame.
    - The `date` column is converted to datetime objects and set as the index.
    - The data is sorted chronologically, which is crucial for time series analysis.
    - The target variable `last_value` (closing price) is selected as the series to analyze (`ts_data`).

## 2. Time Series Visualization
- **Library**: `matplotlib`
- **Action**: A line plot is generated to visualize the historical stock prices over time. This helps in identifying initial trends and outliers.

## 3. Trend, Seasonality, Residual Decomposition
- **Library**: `statsmodels.tsa.seasonal`
- **Action**: 
    - The `seasonal_decompose` function is used to break the time series into three components:
        1.  **Trend**: The increasing or decreasing direction of the data.
        2.  **Seasonality**: Repeating patterns over a fixed period.
        3.  **Residual**: The noise or irregular fluctuations left after extracting trend and seasonality.
    - These components are plotted separately to understand the underlying structure of the data.

## 4. Stationarity Check (ADF Test)
- **Library**: `statsmodels.tsa.stattools`
- **Method**: Augmented Dickey-Fuller (ADF) Test.
- **Action**: 
    - Statistical test to check if the time series is stationary (mean, variance, and covariance do not change over time).
    - **Null Hypothesis**: The series has a unit root (is non-stationary).
    - **Result**: If the p-value is less than 0.05, we reject the null hypothesis and consider the data stationary. Stationary data is easier to model with ARIMA.

## 5. Forecasting with Moving Average
- **Method**: Simple Moving Average (SMA).
- **Action**: 
    - A 30-day rolling window is applied to the data.
    - This smooths out short-term fluctuations to reveal the longer-term trend.
    - It is a simple baseline method for forecasting.

## 6. Forecasting with ARIMA
- **Library**: `statsmodels.tsa.arima.model`
- **Method**: AutoRegressive Integrated Moving Average.
- **Components**:
    -   **AR (p)**: Uses past values for prediction.
    -   **I (d)**: Uses differencing to make the data stationary.
    -   **MA (q)**: Uses past forecast errors.
- **Action**:
    -   The data is split into training (80%) and testing (20%) sets.
    -   An ARIMA model (example order 5,1,0) is fitted on the training data.
    -   Forecasts are generated for the test period.
    -   **Evaluation**: The model is evaluated using RMSE (Root Mean Squared Error) and MAE (Mean Absolute Error).

## 7. Forecasting with LSTM (Deep Learning)
- **Library**: `tensorflow`, `keras`
- **Method**: Long Short-Term Memory (Recurrent Neural Network).
- **Action**:
    -   **Scaling**: Data is normalized to the range [0, 1] using `MinMaxScaler` because neural networks perform better with scaled data.
    -   **Dataset Creation**: The data is restructured into sequences (e.g., using past 60 days to predict the next day).
    -   **Model Architecture**: 
        -   Two LSTM layers with 50 units each (one returning sequences, one not).
        -   Dense layers to output the final prediction.
    -   **Training**: The model is trained on the prepared sequences.
    -   **Prediction**: Predictions are made and then inverse-transformed back to the original price scale.
    -   **Evaluation**: RMSE and MAE are calculated for comparison with ARIMA.

## 8. Conclusion
- The final step involves comparing the error metrics (RMSE/MAE) of both ARIMA and LSTM to decide which model performed better for this specific dataset.
