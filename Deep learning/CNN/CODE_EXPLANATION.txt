## 2. Data Loading and Preprocessing

```python
# Load dataset
df = pd.read_csv('fashion-mnist_test.csv')
print(f"Dataset Shape: {df.shape}")
df.head()
```

**Explanation:**
- `df = pd.read_csv('fashion-mnist_test.csv')`: Reads the comma-separated values (CSV) file into a Pandas DataFrame called `df`.
- `print(...)`: Prints the dimensions of the dataset (Rows, Columns) to verify successful loading.
- `df.head()`: Displays the first 5 rows of the DataFrame to inspect the data structure (usually label column first, followed by pixel columns).

```python
# Extract labels and pixels
labels = df['label'].values
pixels = df.drop('label', axis=1).values

# Reshape to (N, 28, 28, 1)
images = pixels.reshape(-1, 28, 28, 1)

# Normalize pixel values to [0, 1]
images = images.astype('float32') / 255.0

print(f"Images Shape: {images.shape}")
print(f"Labels Shape: {labels.shape}")
```

**Explanation:**
- `labels = df['label'].values`: Selects the 'label' column and converts it to a NumPy array. This holds the target answers (0-9).
- `pixels = df.drop('label', axis=1).values`: Removes the 'label' column to keep only the pixel data, then converts it to a NumPy array.
- `images = pixels.reshape(-1, 28, 28, 1)`: Reshapes the flat row of 784 pixels into a 3D image format: (Number of Images, Height=28, Width=28, Channels=1). The `-1` tells NumPy to automatically calculate the number of images based on the data size.
- `images = ... / 255.0`: Pixel data ranges from 0 (black) to 255 (white). Neural networks perform better with small input values, so we divide by 255.0 to scale everything between 0 and 1. `astype('float32')` ensures efficient memory usage.
- `print(...)`: Confirms the new shapes of our input data (images) and output targets (labels).

```python
train_images, test_images, train_labels, test_labels = train_test_split(
    images, labels, test_size=0.2, random_state=42, stratify=labels
)
```

**Explanation:**
- `train_test_split(...)`: A Scikit-Learn function that shuffles and splits arrays.
- `images, labels`: The data to split.
- `test_size=0.2`: Specifies that 20% of the data should be set aside for testing, while the remaining 80% is for training.
- `random_state=42`: Ensures the split occurs exactly the same way every time this code runs.
- `stratify=labels`: Ensures that the train and test sets have the same proportion of each class (e.g., same % of T-shirts in both) as the original dataset.

## 4. CNN Architecture Construction

```python
def create_model():
    model = models.Sequential([
        # Conv Block 1
        layers.Conv2D(32, (3, 3), activation='relu', input_shape=(28, 28, 1)),
        layers.MaxPooling2D((2, 2)),
        
        # Conv Block 2
        layers.Conv2D(64, (3, 3), activation='relu'),
        layers.MaxPooling2D((2, 2)),
        
        # Flatten and Dense
        layers.Flatten(),
        layers.Dense(128, activation='relu'),
        layers.Dense(10) # 10 classes
    ])
    return model

model = create_model()
model.summary()
```

**Explanation:**
- `def create_model():`: Defines a Python function to encapsulate our model creation logic.
- `models.Sequential([...])`: Initializes a linear stack of layers. Data flows from the top layer to the bottom.
- `layers.Conv2D(32, (3, 3), ...)`: The first Convolutional layer. It learns 32 different filters (feature detectors) of size 3x3. `relu` activation makes the network non-linear. `input_shape` defines what the data looks like coming in.
- `layers.MaxPooling2D((2, 2))`: Reduces the spatial dimensions (Width/Height) by taking the maximum value in every 2x2 window. This reduces computation and controls overfitting.
- `layers.Conv2D(64, ...)`: A second Conv layer with 64 filters. More filters allows learning more complex features.
- `layers.Flatten()`: Converts the 3D output of the convolutional part (e.g., 7x7x64) into a 1D vector (e.g., 3136 numbers) so it can feed into a standard Dense layer.
- `layers.Dense(128, ...)`: A fully connected layer with 128 neurons to interpret the features extracted above.
- `layers.Dense(10)`: The output layer. It has 10 neurons, one for each class (0-9). The raw values here are "logits" (unnormalized scores).
- `model.summary()`: Prints a table showing each layer, its output shape, and the number of trainable parameters.

## 5. Training

```python
model.compile(optimizer='adam',
              loss=tf.keras.losses.SparseCategoricalCrossentropy(from_logits=True),
              metrics=['accuracy'])

history = model.fit(train_images, train_labels, epochs=15, 
                    validation_data=(test_images, test_labels))
```

**Explanation:**
- `optimizer='adam'`: Sets the algorithm that updates the network weights. Adam is an efficient, standard choice.
- `loss=...`: Defines how we calculate error. `SparseCategoricalCrossentropy` is used because our labels are integers (0, 1, 2...). `from_logits=True` tells the function that the output layer didn't use a 'softmax' activation yet (it outputs raw scores).
- `metrics=['accuracy']`: Tracks classification accuracy during training.
- `model.fit(...)`: The command that actually trains the network.
  - `epochs=15`: It will iterate over the entire dataset 15 times.
  - `validation_data=(...)`: After every epoch, it checks performance on the test data to see how well it's generalizing.
  - Returns a `history` object containing loss/accuracy values for every epoch.

## 9. Model Improvement (Augmentation)

```python
data_augmentation = tf.keras.Sequential([
  layers.RandomFlip("horizontal", input_shape=(28, 28, 1)),
  layers.RandomRotation(0.1),
  layers.RandomZoom(0.1),
])
```

**Explanation:**
- `tf.keras.Sequential`: Defines a pipeline specifically for preprocessing.
- `layers.RandomFlip("horizontal")`: Randomly flips images left-to-right during training. This teaches the model that a shoe pointing left is the same object as a shoe pointing right.
- `layers.RandomRotation(0.1)`: Randomly rotates images by up to 10%.
- `layers.RandomZoom(0.1)`: Randomly zooms in or out by 10%.
- **Note**: These layers are only active during `model.fit()`. They do nothing during prediction/testing.

```python
def create_improved_model():
    model = models.Sequential([
        data_augmentation,
        # ... Conv Layers ...
        layers.Dropout(0.25),
        # ... More Layers ...
        layers.Dropout(0.5), 
        # ... 
    ])
```

**Explanation:**
- `layers.Dropout(0.25)`: Randomly "turns off" 25% of the neurons in the layer during each training step. This prevents the network from relying too heavily on any single neuron (memorization) and forces it to learn robust features. We add this to reduce overfitting.
